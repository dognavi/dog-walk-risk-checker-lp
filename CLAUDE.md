# CLAUDE.md

このファイルは Claude Code がこのリポジトリで作業する際の指針です。

## プロジェクトの目的

愛犬の散歩に潜む危険（熱中症・アスファルト熱・紫外線など）を可視化する「散歩危険度チェッカー」ツールを紹介する静的LP。訪問者が思わずツールを試したくなるデザイン・コピーが最重要目的。

## 技術スタック

- HTML5 / CSS3 / Vanilla JavaScript のみ。ビルドツール・フレームワークは導入しない（LPの軽量さ・表示速度を優先）。
- 画像は `images/` に配置し、WebP等軽量フォーマットを優先する。

## ディレクトリ構成

- `index.html` — LP本体（全セクションを1ページに集約）
- `css/style.css` — スタイル一式
- `js/script.js` — スクロール演出・危険度デモなどのインタラクション
- `images/` — 画像素材

## デザイン方針

- ターゲットは犬を飼う一般ユーザー。専門用語は避け、感情に訴える言葉（「大丈夫だと思っていませんか？」等）とビジュアルで危険性を直感的に伝える。
- ファーストビューで「危険度スコア」のイメージを見せ、CTA（ツールを使う）への導線を複数箇所に配置する。
- 暖色・警告色（危険度が上がると赤系に変化するグラデーション等）を効果的に使い、「今すぐ確認したい」という行動を促す。
- モバイルファーストでレスポンシブに実装する（散歩前にスマホで見られる想定）。

## コーディング規約

- セマンティックなHTML（`section`, `header`, `main`, `footer` 等）を使う。
- CSSクラス名はBEM風（`block__element--modifier`）で統一する。
- JSはモジュール分割せず単一ファイルで十分な規模だが、機能ごとに関数を分けて可読性を保つ。
- アクセシビリティ（altテキスト、コントラスト比、フォーカス可視化）に配慮する。

## よく使うコマンド

- ローカル確認: `python -m http.server 8000` などでルートを配信し、`index.html` をブラウザで開く。
- 自動テスト・Lintは現時点で未導入（静的LPのため）。

## 注意事項

- `.env` 等の秘密情報ファイルは読み取らない・コミットしない。
- 外部API（気象データ等）と連携する場合はAPIキーを`.env`で管理し、`.env.example`にキー名のみ追記する。

※ コードが揃ったら `/init` で本ファイルを更新・拡充できます。
