# 散歩危険度チェッカー紹介LP 実装計画

## 前提・確定事項（ユーザー指定・変更しない）

- **目的**: 見た人にツールを使ってもらう（CTAボタンを押してもらう）
- **ターゲット**: 犬の飼い主さん全般
- **トーン**: やさしく語りかける口調・雰囲気
- **メインコピー**: 「その散歩、今日の暑さは大丈夫？」
- **サブコピー**（レビュー反映で変更）: 「気温・湿度・日ざしから、愛犬の散歩の危険度がひと目で分かります。」（「リスク」→「危険度」でツール名と表記を揃え、「判定します」→「ひと目で分かります」で飼い主目線・やさしいトーンに調整）
- **ボタン文言**: 「今すぐチェックする」
- **ボタン直下のマイクロコピー**（レビュー反映・新規追加）: 「無料・登録不要・インストール不要」といった趣旨の1行を各CTAボタンの直下に添え、行動のハードルを下げる。具体的な文言はPhase 1でドラフトしユーザー確認を取る。
- **ボタンリンク先**: `https://dognavi.github.io/dog-walk-risk-checker/?ref=lp`
  - ベースURL（`https://dognavi.github.io/dog-walk-risk-checker/`）は実在確認済み（2026-07-10 WebFetchで確認）。「散歩危険度チェッカー」というWebアプリで、位置情報から気温・湿度・日射量等を取得しリスク判定を行う、位置情報非保存・免責事項ありの正規ツール。無料・認証不要・サーバーレス。
  - レビュー反映：LP経由の流入を後で判別できるよう `?ref=lp` をクエリパラメータとして付与する。
- **構成（上から4ブロック、この順序を厳守）**:
  1. ファーストビュー（メインコピー＋サブコピー＋CTAボタン＋マイクロコピー）
  2. なぜチェックが必要か（レビュー反映：理由づけ込みで3点に確定）
     - 湿度: 犬は汗をかけず、ハアハアという呼吸（パンティング）でしか体温を下げられないため、湿度が高いと体温調節が効きにくくなる、という仕組みまで説明する。
     - 地面との距離: 犬は地面に近いぶん、照り返しや地面の熱を受けやすい。
     - 我慢してしまう犬の性質: 犬は不調を言葉で伝えられず我慢してしまいがちなので、飼い主が先回りして確かめてあげる必要がある（＝ツールを使う理由に直結させる）。
     - ※注意（レビュー反映）: 数字や獣医学的な内容は一般に広く言われている範囲にとどめ、創作した数値や断定的な表現は使わない。盛らない。
  3. このツールで何が分かるか（レビュー反映：機能列挙から「便益＋手軽さ」訴求に変更）
     - 便益を前面に: 「今日、散歩に出て大丈夫かがひと目で分かる」
     - 手軽さを3ステップで見せる: 「開く → 位置情報を許可 → 結果が出る」
     - 「ツールのスクショ」プレースホルダーはそのまま維持し、判定バッジ等のモックは作らず後工程で実物のスクショを貼る（レビュー反映）。
  4. 締め（コピーの再訴求＋CTAボタン再掲＋マイクロコピー）
- **画像**: 今回は実画像を入れない。「ここに犬の画像」「ここにツールのスクショ」という配置プレースホルダーのみ用意し、後工程ではめ込む。
- **既存資産**: `index.html` / `css/style.css` / `js/script.js` のビルドレスな静的サイト。`README.md`・`CLAUDE.md` に技術スタック・ディレクトリ構成・デザイン方針の記載あり（BEM命名、セマンティックHTML、モバイルファースト、暖色＋警告色グラデーション）。

## 確定コピー（ユーザー承認済み・2026-07-10）

Phase 1のHTML実装はこのコピーをそのまま反映する。以降の変更は再度ユーザー承認を要する。

### ブロック2:なぜチェックが必要か

**セクション見出し(H2)**
> その暑さ、愛犬には大きな負担かもしれません

**気づき①**
- 見出し(H3): 湿度が高いと、体温をうまく下げられない
- 本文: 犬は人のように汗をかいて体を冷やすことができません。ハアハアという呼吸(パンティング)で体温を逃がしているため、湿度が高い日は、その呼吸による体温調節がうまく働きにくくなると言われています。気温だけでなく、湿度もあわせてチェックしてあげることが大切です。

**気づき②**
- 見出し(H3): 地面に近い分、照り返しの影響を受けやすい
- 本文: 犬は私たちよりずっと地面に近い場所を歩いています。そのため、アスファルトなどからの照り返しや地面の熱を、人よりも受けやすいと言われています。「今日はそこまで暑くない」と人が感じる日でも、犬の足元は、もっと暑くなっているかもしれません。

**気づき③**
- 見出し(H3): 犬は、つらくても我慢してしまいます
- 本文: 犬は言葉で「暑い」「つらい」と伝えることができません。多少無理をしていても、飼い主さんと一緒にいたい一心で、そのまま歩き続けてしまうことがあります。だからこそ、愛犬が我慢する前に、飼い主さんの方から「今日は大丈夫かな?」と確かめてあげたいですね。

### ブロック3:このツールで何がわかるか(便益+手軽さ)

**セクション見出し(H2)**
> 今日、散歩に出て大丈夫かが、ひと目でわかります

**サブ文**
> 難しい入力は必要ありません。今いる場所の気温・湿度・日ざしから、散歩の危険度をすぐに確認できます。

**3ステップ**
1. **開く** ― 「散歩危険度チェッカー」を開きます
2. **位置情報を許可する** ― 今いる場所の空の状態を取得します
3. **結果を見る** ― 今日の散歩の危険度がすぐに表示されます

（この下に「ここにツールのスクショ」プレースホルダーを配置）

## 計画上の判断（曖昧点の扱い）

以下は指示に明記がなかったため、実装時のデフォルト方針として計画に含める。異論があればPhase開始前に指摘してほしい。

- **CTAリンクの開き方**: 外部の別ツールに遷移するため `target="_blank" rel="noopener noreferrer"` で新規タブに開く（LPのコンテキストを保持し離脱を防ぐため）。
- **既存READMEにあった「ヒーローのインタラクティブ危険度デモ」構想**: 今回の確定コピー（メインコピー＋サブコピー＋ボタンのみ）を優先し、今回は実装しない。Phase 4で README の TODO を現状に合わせて更新する。
- **トーンと配色の整合**: CLAUDE.mdの「警告色（赤系グラデーション）」は保持しつつ、「やさしく語りかける」トーンに合わせて赤を主色にはせず、暖かみのあるベースカラー（オフホワイト・生成り色・淡いオレンジ等）に、リスクを示す差し色として警告色を要所（ボタンやアイコン等）に限定的に使う方針とする。

## 進め方（重要）

**このPhaseごとに実装 → スクリーンショット/検証結果を提示 → ユーザー承認 → 次のPhaseへ進む。** 全Phase一括実装はしない。承認が出るまで次のPhaseのコードは書かない。

**追加ゲート（レビュー反映）**: ブロック2・3は指定コピーそのものではなく新規に書き起こす文章のため、Phase 1でHTMLに実装する**前**に、ドラフトのテキスト（見出し・本文・マイクロコピー案）だけをユーザーに提示し承認を得る。指定コピー（メインコピー・サブコピー・ボタン文言）の一致確認とは別に、この新規コピーの承認を独立したチェック項目として扱う。

- CTAボタンをブロック1とブロック4の2箇所に設置し、`https://dognavi.github.io/dog-walk-risk-checker/?ref=lp` へのリンク（新規タブ、`target="_blank" rel="noopener noreferrer"`）を張り、直下にマイクロコピーを添える。

## Phase 3: インタラクション・アクセシビリティ仕上げ

### 作業内容
- CTAボタンのフォーカスリング（キーボード操作時の視認性）を実装する。
- 画像プレースホルダーに `aria-label` で説明を付与し、将来画像を差し込む際に迷わないようにする（Phase 1で装飾用の `aria-hidden` を使わない方針にしているため、矛盾なく追加できる）。
- 必要に応じて軽いスクロールアニメーション（`IntersectionObserver`によるフェードイン等）を `js/script.js` に実装する（過剰な演出は避け、やさしいトーンを崩さない範囲に留める）。
- OGP用メタタグ（`og:title` / `og:description`）を追加し、SNSシェア時にコピーが正しく表示されるようにする。

### 完了条件（検証ゲート）
- `preview_click` でTabキー相当のフォーカス移動を確認できない場合は `preview_eval` でフォーカス可能要素の並びとフォーカススタイルの有無を確認する。
- `preview_click` でCTAボタンを実際にクリックし、新規タブで `https://dognavi.github.io/dog-walk-risk-checker/?ref=lp` に遷移する（または `window.open` が正しいURLで呼ばれる）ことを確認する。
- `preview_console_logs` でJSエラーが出ていないことを確認する。
- OGPタグがhead内に存在し、値がコピー内容と一致することを確認する。

## データソース・依存ライブラリ

- 本LP自体は外部API・外部ライブラリを一切使用しない（HTML5/CSS3/Vanilla JSのみ、ビルドツールなし）。無料/有料の制約やAPIキーは不要。
- 唯一の外部依存はCTAリンク先の `https://dognavi.github.io/dog-walk-risk-checker/?ref=lp`（既存の別ツール、実在確認済み、無料・認証不要。`?ref=lp` はLP経由の流入判別用クエリパラメータ）。このツール内部がOpen-Meteo APIを使用しているが、それは先方の実装範囲であり本LP側の依存関係には含まれない。
- 画像素材は今回のスコープ外（プレースホルダーのみ）。後工程で実画像/スクリーンショットを用意する必要がある（未定・別タスク）。
